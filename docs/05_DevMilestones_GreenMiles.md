## 1. 문서 개요

본 문서는 그린마일즈(GreenMiles) MVP를 단계별로 실행할 수 있도록 7개 개발 마일스톤과 각 단계의 완료 기준, 테스트 기준을 정의한다. 1차 개발은 대형마트 실제 POS 연동이 아니라 가상 POS와 QR claim으로 제한하며, 블록체인은 실제 토큰 발행이 아니라 MRV batch hash 앵커링으로 제한한다. 모든 마일스톤은 1~2주 안에 완료 가능한 단위로 구성되었다.

---

## 2. 전체 개발 전략

| 전략 | 설명 |
|---|---|
| 작게 시작 | 가상 POS, QR, 영수증 인증을 중심으로 20~30개 SKU, 300건 구매 인증 데이터를 목표로 작게 검증 |
| 핵심 루프 우선 | "구매 인증 → 리워드 계산 → 지갑 반영 → 대시보드 확인"을 가장 먼저 완성 |
| 신뢰 구조는 후순위 | MRV 배치/블록체인 앵커링은 핵심 루프 완성 이후 추가 |
| 코드 에이전트 활용 | Codex에게 기능 단위로 작업을 분배하여 개발 속도 확보 |
| 표현 통제 | 모든 화면/문서에서 법정 탄소배출권 표현을 금지하고 단계별 상태 표현을 사용 |

---

## 3. MVP 범위

- 상품별 탄소 데이터 레지스트리(SKU 등록/승인)
- 가상 POS 구매 이벤트 생성 및 QR claim
- 영수증 업로드 및 관리자 수동 검증
- 푸드 마일리지 기반 리워드 계산 엔진
- 소비자 리워드 지갑
- 유통사 ESG 대시보드(기간/매장/SKU/카테고리 필터)
- MRV 배치 생성 및 블록체인 테스트넷 해시 앵커링
- 관리자 운영 화면(상품 승인, 영수증 검토, 배출계수/정책 관리, MRV 상태 관리)

---

## 4. 제외 범위

- 실제 대형마트 POS 시스템 연동
- 법정 탄소배출권의 실제 발행/거래/상쇄
- 블록체인 메인넷 배포
- 완전 자동화된 OCR 파이프라인(보조 기능으로만 포함)
- 네이티브 모바일 앱
- VCM 파트너와의 실제 정산 계약 이행

## 9. 5차 마일스톤: 영수증 업로드 / 관리자 검증

| 항목 | 내용 |
|---|---|
| 목표 | QR 환경이 없는 매장에서도 구매 인증이 가능하도록 보완한다 |
| 구현 기능 | 영수증 업로드, (선택) OCR 보조 추출, 관리자 검토/승인/반려 화면 |
| 입력 데이터 | TRD: GreenMiles 14번 섹션, User Flow 7번 플로우 |
| 산출물 | 영수증 업로드 UI, 관리자 검토 큐 화면 |
| 완료 기준 | 영수증 업로드부터 승인까지의 흐름이 정상 동작하고, 승인된 건이 리워드 계산까지 연결될 것 |
| 테스트 기준 | 반려 시 사유 전달 및 재제출 플로우 테스트 통과 |
| 예상 리스크 | 영수증 위변조 가능성, OCR 정확도 부족 |
| 다음 단계 | 6차 마일스톤(MRV batch/블록체인 앵커링) 착수 |

## 14. Codex 병렬 작업 분리 방안

| 작업 트랙 | 담당 영역 | 의존성 |
|---|---|---|
| 트랙 A | DB 스키마/RLS/Auth (1차) | 선행 작업, 모든 트랙의 기반 |
| 트랙 B | 가상 POS/QR claim/리워드 엔진 (2~3차) | 트랙 A 완료 후 시작 |
| 트랙 C | 유통사 대시보드 UI (4차) | 트랙 A 완료 후 트랙 B와 병렬 가능 |
| 트랙 D | 영수증 업로드/관리자 검증 (5차) | 트랙 A 완료 후 트랙 B와 병렬 가능 |
| 트랙 E | MRV batch/블록체인 앵커링 (6차) | 트랙 B 완료 후 시작 |
| 트랙 F | 테스트/데모/README (7차) | 모든 트랙 완료 후 통합 |

트랙 C와 D는 트랙 B의 핵심 데이터 모델만 확정되면 병렬로 Codex에게 위임 가능하다.

## 15. 개발 우선순위

1. 데이터 모델 및 권한(1차) — 모든 기능의 기반
2. 구매 인증(2차) — 데이터 수집의 출발점
3. 리워드 엔진/지갑(3차) — 소비자 가치 제공의 핵심
4. 유통사 대시보드(4차) — B2B 가치 제공 및 수익화 근거
5. 영수증 업로드(5차) — 인증 채널 확대
6. MRV/블록체인(6차) — 신뢰성 확보
7. 테스트/데모(7차) — 검증 및 파일럿 준비

| 영수증 업로드~승인/반려 플로우 정상 동작 | ☐ |
| MRV 배치 생성 및 상태 전이 정상 동작 | ☐ |
| 블록체인 테스트넷 앵커링 성공/실패 케이스 모두 처리 확인 | ☐ |
| 개인정보/구매 상세가 온체인에 저장되지 않는지 확인 | ☐ |

## 17. 데모 시나리오

1. 관리자가 SKU 5개를 사전 등록·승인해 둔다.
2. 유통사 계정으로 로그인하여 가상 POS 구매 이벤트를 생성하고 QR을 발급한다.
3. 소비자 계정으로 QR을 스캔하여 구매를 인증하고, 지갑에서 즉시 적립된 포인트와 예상 탄소 기여도를 확인한다.
4. 다른 소비자 계정으로 영수증을 업로드하고, 관리자가 이를 승인하여 동일하게 리워드가 적립되는 과정을 보여준다.
5. 유통사 계정으로 돌아가 대시보드에서 누적된 판매량, 리워드 발행량, 재구매율 데이터를 확인한다.
6. 관리자가 누적된 리워드 이벤트를 MRV 배치로 묶고, 테스트넷 앵커링을 실행해 "블록체인 기록 완료" 상태로 전환되는 과정을 시연한다.

## 핵심 요약

본 문서는 GreenMiles MVP를 7개 마일스톤(세팅/DB/인증 → 상품·POS·QR → 리워드·지갑 → 대시보드 → 영수증 → MRV·블록체인 → 테스트·데모)으로 나누고, 각 마일스톤의 목표·산출물·완료 기준·테스트 기준·리스크를 정의했다. 4주 MVP 일정과 8주 확장 일정을 구분했으며, Codex에게 병렬로 위임 가능한 6개 작업 트랙과 QA 체크리스트, 데모 시나리오를 제공한다. 1차 개발은 가상 POS/QR로 제한되며, 블록체인은 MRV batch hash 앵커링으로 제한된다.
